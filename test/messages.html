XXXXXERROR CODEx1

Description: Memory access error: the content in a memory address is not as expected.
Possible source: peek()
poke()
Hints:
  • Did you peek/poke into the right memory address?
  • Did you peek/poke byte, word, and long properly?

XXXXXERROR CODEx2

Description: Screen output error: incorrect characters printed on screen.
Possible source: cls()
output_string()
output_char()
printnum()
Hints:
  • Did you clear the screen and place the cursor at the top left corner in cls()?
  • Did you move the cursor every time you printed a character?
  • Do you move the cursor to the next line when you print '\n'?
  • Did you use poke_b(), not poke_w() or poke_l(), to print characters onto the screen?
  • Did you calculate the memory address correctly? (Each visible character is represented by two bytes. The first byte is used for the character and the second byte is used for the attribute. Each line can display 80 visible characters, and there are 25 lines.)
  • Did you scroll the screen up when the cursor reached the bottom of the screen?
  • In printnum(), did you print '-' for negative numbers?
  • Did you convert the number to the correct base?
  • Did you remember to put padding characters when required?
  • When printing hexadecimal numbers, did you use upper case or lower case characters as required?

.

XXXXXERROR CODEx19

Description: Ready queue error: the double linked list structure of the ready queues is not maintained correclty.
Possible source: add_ready_queue()
remove_ready_queue()
Hints:
  • Did you set a process's "next" and "prev" (in the PCB entry) correctly when you add it to the double linked list?
  • When you add a process to a ready queue, did you reset "next" and "pre" of the previous process and those of the next process?
  • When you remove a process from a ready queue, did you also reset "next" and "pre" of the previous process and those of the next process?
  • Is the double linked list Round-Robin? ("next" of the last process should point to the first process, and "pre" of the first process should point to the last process.)
  • Did you set ready_queue[priority] to null when the last process is removed from the ready queue of that priority?
  • When there is only one process on a ready queue, are its "next" and "pre" pointing to itself?
